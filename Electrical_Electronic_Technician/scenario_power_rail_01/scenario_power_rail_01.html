Diagnostic Steps
1
Verify Power Input at TP1
Multimeter reads 3.30V DC at TP1. Source is nominal — fault is downstream.
V_input = 3.30V  ✓  (Spec: 3.3V ± 5% → 3.135V – 3.465V)
2
Measure MCU VCC with Multimeter (TP3)
Multimeter reads 3.18V DC at TP3 — 120mV below input. DC alone doesn't reveal the resets, but confirms resistive loss on the net.
ΔV = 3.30 − 3.18 = 0.12V  |  R_eff = ΔV / I = 0.12 / 0.15A = 0.8Ω
Scope needed — multimeter averages out fast transients.
3
Probe TP3 with Oscilloscope — Find Transients
Scope reveals periodic sags crossing the 2.97V brownout threshold at ~3ms intervals, correlating with MCU active cycles drawing peak current.
V_sag = 3.3 − I × (R_trace + ESR_C2) = 3.3 − 0.15 × (0.8 + 2.0) = 2.88V
C2 (100nF) has high ESR (2.0Ω). It cannot supply transient charge fast enough during current spikes — the rail collapses to 2.88V, triggering brownout reset.
★
Root Cause & Fix
ROOT CAUSE: Bypass capacitor C2 has degraded ESR (2.0Ω vs. spec <0.1Ω for MLCC). High ESR prevents C2 from sourcing transient current during MCU load steps, causing brownout reset.

FIX:
  • Replace C2 with low-ESR MLCC 100nF (X5R or X7R dielectric)
  • Add 10µF MLCC bulk cap at U1 VCC pin
  • Audit +3V3 trace width — 0.8Ω suggests under-spec or damaged trace
CAD Note (Altium Designer / OrCAD / Eagle):
Run DRC — verify C2 footprint is MLCC, not electrolytic.
Use PCB trace width calculator: IPC-2221 requires ≥ 0.25mm (1oz Cu) for 150mA.
In Altium: Tools → PCB Inspector → inspect +3V3 net trace impedance.

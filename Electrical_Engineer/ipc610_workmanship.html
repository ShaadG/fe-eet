IPC-A-610H — Three Product Classes Section 1.4
Class Name Description Typical Applications Criteria Strictness
Class 1 General Electronics Products where the primary requirement is function of the completed assembly. Cosmetic imperfections acceptable. Consumer electronics, toys, non-critical appliances Most permissive — focus on function only
Class 2 Dedicated Service Electronics Products where continued performance and extended life is required. Uninterrupted service preferred but not critical. Communications equipment, industrial controls, commercial computers Moderate — cosmetic defects allowed if function is unaffected
Class 3 High Reliability Electronics Products where continued performance is critical. Equipment downtime is not acceptable. Harsh environment use. Military, aerospace, medical life-support, defense avionics Most strict — defects that are Class 2 acceptable may be Class 3 defects
📋 Workflow tie-in: Before writing an NCR, confirm which class the board is built to — it is specified in the work order or customer contract. A solder joint condition that PASSES Class 2 may be a DEFECT for Class 3. The job description specifies defense and aerospace — that means Class 3 by default.

The Three Condition Categories — Core Vocabulary

Every condition in IPC-A-610H falls into one of three categories. Knowing these three terms cold is the single most important thing to demonstrate in an interview about the standard.

Category Definition Action Required NCR Required?
Target Condition Near perfect — the ideal workmanship condition. Desirable but not always achievable in production. None — document as pass. No
Acceptable Condition Not perfect, but the assembly will function reliably in its intended environment. Within the allowable criteria for its class. None — ship as-is. No
Process Indicator Not a defect — the assembly may still function — but indicates a process that is not well controlled and may produce defects in the future. Requires process investigation. Flag for process engineering review. Do not automatically reject. Sometimes — per customer contract
Defect A condition that is not acceptable per the applicable class. The assembly must not be shipped in this condition. Requires disposition (rework, repair, scrap, or customer waiver). Write NCR. Dispositioned by engineering before release. Yes — always
📋 Interview tip: When asked "what would you do if you found a cold solder joint during inspection?" — the correct answer is: "Determine which class the board is built to, look up the condition in IPC-A-610H, determine if it is a defect or process indicator for that class, and write an NCR if it qualifies as a defect. I would not rework it myself without engineering authorization."

Visual Solder Joint Reference — Condition Examples
TARGET
Smooth concave fillet. Full wetting. Shiny surface. Component well-centered on pad.
ACCEPTABLE
Slightly convex fillet but still wetted to pad and component. No exposed base metal.
PROCESS INDICATOR
Dull, grainy surface (cold joint indicator). Function intact but process needs investigation.
DEFECT — Insufficient
Very thin solder layer. Pad not fully covered. High-resistance joint. NCR required.
DEFECT — Bridge
Solder spans two pads. Electrical short. NCR required. Wick with desoldering braid.
DEFECT — Tombstone
One end lifted off pad. Open circuit on that end. Common with small chip components. NCR required.
📋 Workflow tie-in: When handing a board to the rework tech, describe the defect using IPC-A-610H language: "Class 3 defect — insufficient solder on R14, pad 2 — less than 25% fillet height." This gives the rework tech exactly what they need and makes your NCR traceable.

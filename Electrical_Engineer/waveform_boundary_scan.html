EE I Application — Power-On Bring-Up

During morning power-on bring-up, after confirming DC rail levels with the multimeter (±5% check), switch the oscilloscope to AC coupling to isolate any ripple sitting on the DC supply. A clean rail shows a flat line or at most a small periodic ripple at the switching regulator's frequency. This sine reference represents that ripple waveform when it is within spec.

  • Period: 2 divs × 0.5 ms/div = 1 ms → f = 1/0.001 = 1 kHz
  • Vpp: 3 divs × 2 V/div = 6 V
  • Vrms: Vpeak / √2 = 3.0 / 1.414 = 2.12 V
Workflow connection (Mid-Morning): This is step 2 of the bring-up sequence in workflow_EE1_daily_weekly.md — "Oscilloscope checks on clock signals, PWM outputs, and communication buses." A clean waveform here is the green light to proceed to boundary-scan.

EE I Application — Fault Isolation

This is the waveform you would observe on the 3.3V power rail in Scenario 01 — a bypass capacitor (C2) with degraded ESR is failing to filter switching noise. The multimeter reads a normal DC average (~3.18V) and would not flag this fault. Only the oscilloscope reveals the noise component.

  • Switch to AC coupling and reduce V/div to 100 mV/div to measure noise amplitude accurately.
  • If the noise is periodic at the regulator's switching frequency → decoupling cap failure.
  • If the noise is random → EMI / crosstalk from adjacent trace or switching element.
  • ESR test: power off, remove C2, measure with LCR meter. ESR > 0.5Ω on a small MLCC = replace.
Boundary-scan connection: A noisy power rail that causes logic threshold violations will generate false boundary-scan failures — a good solder joint reads as an open because the captured logic level is indeterminate. Always verify power rail quality with the oscilloscope before running a boundary-scan test. Power integrity is a prerequisite, not a follow-up.

EE I Application — Clock & Logic Signal Verification

Before running a boundary-scan, verify that the TCK (test clock) and TMS (test mode select) lines are switching correctly at the expected levels. A 3.3V CMOS-clean square wave here confirms the JTAG interface is electrically sound before you interpret any scan results.

  • VOH ≥ VIH(min): 3.3V output must exceed the receiving device's minimum input HIGH threshold (typically 0.7 × VDD = 2.31V for 3.3V CMOS).
  • VOL ≤ VIL(max): Output LOW must stay below 0.3 × VDD = 0.99V.
  • Rise time: Zoom to 50 ns/div — slow edges on TCK reduce scan clock reliability; check for excessive capacitive loading on the JTAG header.
  • Duty cycle: TCK should be close to 50% for reliable data capture on both rising and falling edges.
Boundary-scan connection: TCK is the clock that drives every bit shift in the scan chain. If TCK signal integrity is degraded — slow edges, reduced VOH, ringing — the Toolstick may miss clock edges and shift corrupted data, producing false fault reports. Probe TCK on the JTAG header first.

Reading the JTAG Timing Diagram

The four JTAG signals each have a specific role. When probing with an oscilloscope, set up 4-channel capture with all channels referenced to the same board ground point.

  • TCK (Test Clock): Free-running clock generated by the Toolstick. All data shifts occur on the rising edge of TCK. VOH must meet 3.3V CMOS thresholds.
  • TMS (Test Mode Select): Controls the TAP state machine. The pattern shown (1-1-0-0-0-1-1-0-0...) steps through: Reset → Run-Test/Idle → Shift-DR → Update-DR → Idle.
  • TDI (Test Data In): Serial data shifted into the first device in the chain on each rising TCK edge. Must be stable before TCK rises (setup time ≥ 25 ns typical).
  • TDO (Test Data Out): Serial data shifted out of the last device in the chain. Valid after the falling edge of TCK (propagation delay ≤ 15 ns typical). The Toolstick samples TDO on TCK's falling edge.
What to look for on the scope: TDO should mirror TDI shifted by N-bits (where N = total scan chain register length). If TDO stays static — stuck HIGH or stuck LOW — the scan chain is broken: a device is missing, unpowered, or has a failed TDI→TDO path. This is a boundary-scan chain fault, not a net fault.

Physical Connections on the PCB

The scan chain daisy-chains every JTAG-compliant device on the board in series. TDI enters IC1, passes through IC1's boundary-scan register, exits as IC1's TDO — which is then wired to IC2's TDI, and so on. The Toolstick's TDO input connects to the final device's TDO.

Toolstick (Host PC via USB)
   │
   ├── TCK ──────────────────────────────────────► all devices
   ├── TMS ──────────────────────────────────────► all devices
   │
   ├── TDI ──► [IC1: FPGA] ──TDO→TDI──► [IC2: MCU] ──TDO→TDI──► [IC3: DDR] ──TDO──┐
   │                                                                                 │
   └── TDO ◄─────────────────────────────────────────────────────────────────────────────────────┘
  • IDCODE phase: Toolstick sends IDCODE instruction. Each device returns its 32-bit ID. IC1=0x0364A093 (Xilinx), IC2=0x4BA00477 (ARM Cortex-M), IC3=0x01B30043 (JEDEC DDR).
  • EXTEST phase: Toolstick loads test vectors. IC1 drives its output pins; IC2 and IC3 capture what arrives on their connected input pins. Toolstick compares captured vs. expected.
  • Fault report: Output is net-level — e.g., IC1.PIN_A14 → IC2.PIN_PA0 : OPEN (driven=1, captured=0).
Oscilloscope role here: If the IDCODE phase fails (no devices detected), probe TDI and TDO with the scope. TDI should show the clock-synchronous data pattern. If TDO is flat, the chain is broken. Scope probe on the TDO of each device identifies exactly which device's output is stuck.
Tool Comparison — When to Use the Scope vs. Boundary-Scan EE I Decision Guide
Oscilloscope
  • Verifying power rail DC levels and ripple
  • Checking clock signal integrity (rise time, VOH, VOL)
  • Observing communication bus activity (SPI, I²C, UART)
  • Catching transient events: brownouts, glitches, ringing
  • Confirming JTAG signal integrity before boundary-scan
  • Measuring noise amplitude and frequency
Boundary-Scan (JTAG / Toolstick)
  • Detecting solder joint opens and shorts at pin level
  • Verifying component presence (IDCODE check)
  • Testing interconnects on fine-pitch / BGA components with no probe access
  • Structural test after assembly — before functional test
  • Fault isolation to specific net name and pin reference
  • Programming Flash / FPGA bitstream via JTAG
Rule of thumb: Scope first, boundary-scan second. The oscilloscope confirms the electrical environment (power rails, clock quality) is valid. Only then does boundary-scan produce reliable structural test results. A failed boundary-scan on a noisy board is ambiguous — fix the power integrity issue first.

Test sequence for a board that fails boundary-scan: (1) Confirm all power rails are within ±5% — multimeter. (2) Scope the 3.3V rail in AC coupling — confirm <50 mV p-p ripple. (3) Scope TCK and TMS on the JTAG header — confirm clean 3.3V CMOS square waves. (4) Scope TDI and TDO simultaneously — confirm TDO responds to TDI pattern. (5) If all four pass → trust the boundary-scan fault report as a real structural fault. (6) If any fail → fix the electrical issue and re-run before logging an NCR.
CAD Notes — Altium Designer / OrCAD / Eagle Design-for-Testability

JTAG Header Placement (Altium Designer)

Before a board goes to manufacture, verify the following in the Altium board file:

Altium Designer — DFT Checklist
  • JTAG connector footprint: Place a dedicated 10-pin (2×5, 1.27mm pitch) or 20-pin Cortex Debug header. Assign net labels: TCK, TMS, TDI, TDO, nTRST, VCC_3V3, GND.
  • Net inspector — scan chain: Tools → Net Inspector → verify TDO of IC[n] connects to TDI of IC[n+1] with no stubs or T-junctions. Stubs on TCK degrade signal integrity at high scan frequencies.
  • DRC rule — series termination: Add a 33Ω series resistor on TCK near the driver if the trace is longer than 5 cm. Check DRC clears on the termination resistor net.
  • Layer stack: Route JTAG signals on an internal signal layer referenced to a ground plane for controlled impedance. Target 50Ω single-ended for high-speed JTAG (> 10 MHz TCK).
  • IPC-2221 trace width: JTAG signals are low-current (≤ 10 mA). Minimum trace width for signal integrity, not current capacity — 0.1 mm (4 mil) minimum per IPC-2221 for internal layer; 0.15 mm for external.

Oscilloscope Probe Points — Design Requirement

Altium Designer — Test Point Footprints
  • Add TP (test point) footprints on each power rail: VDD, 3V3, 1V8, GND. Use a 1mm round SMD pad — large enough for an oscilloscope probe tip or multimeter probe.
  • Add test points on JTAG signals: TCK, TDI, TDO, TMS — allows scope probing of boundary-scan signals without soldering a clip to fine-pitch IC pins.
  • Silkscreen label each TP with the net name — TP1/3V3, TP2/GND, TP3/TCK. This is what appears in the scenario_power_rail_01 HTML file as TP1, TP2, TP3.
  • Check fabrication clearances: Test point pads must clear adjacent component courtyard by ≥ 0.2mm (Altium DRC: Clearance rule between TestPoint and Component).

BSDL File Verification — Pre-BOM Freeze

OrCAD / Altium — BOM Verification Step
  • For every JTAG-compliant IC in the BOM, verify a BSDL file is available from the manufacturer before the BOM is frozen. No BSDL = no boundary-scan test for that device.
  • BSDL files are downloaded from the manufacturer's website (TI, Xilinx, NXP, STMicro all publish them). Store in project/bsdl/ folder alongside the Altium project files.
  • Validate BSDL syntax with the Toolstick software before board manufacture — a BSDL error discovered at test stage costs an ECO and schedule delay.
  • Non-JTAG devices (passive, simple analog ICs) are tested structurally through their neighboring JTAG device's boundary-scan register — the neighbor drives the net, the non-JTAG device is a passive element on that net.
